# E18 프로필 수정 API 및 화면 스펙

- 작성일: 2026-03-18
- 대상 작업: `E18-1`
- 관련 API: `PATCH /api/users/me`
- 관련 화면: `frontend/src/pages/Profile.jsx`

## 목적
- 사용자가 닉네임과 주소를 직접 수정할 수 있게 한다.
- 저장 직후 프로필 화면뿐 아니라 헤더와 채팅 등 전역 사용자명 표시도 함께 갱신한다.

## 수정 범위
- 닉네임
- 주소
- 선택적 위치 좌표
  - 이번 화면에서는 좌표 직접 수정 UI를 열지 않는다.
  - API는 좌표 쌍을 선택적으로 받을 수 있다.

## API 요청 형식
```json
{
  "name": "새닉네임",
  "address": "서울 송파구",
  "latitude": 37.500000,
  "longitude": 127.100000
}
```

## 검증 규칙
- `name` 필수
- `address` 필수
- `latitude`, `longitude`는 둘 다 보내거나 둘 다 생략해야 한다.
- 다른 사용자가 이미 쓰는 닉네임이면 충돌 에러를 반환한다.

## 저장 정책
- 주소가 바뀌고 좌표를 함께 보내지 않으면 기존 저장 좌표를 초기화한다.
- 닉네임 또는 주소만 바뀐 경우에도 수정 응답은 최신 프로필 전체를 반환한다.
- 프론트는 저장 성공 후 `AuthContext` 프로필을 다시 동기화해 전역 사용자명을 맞춘다.

## 화면 동작
- 기본은 읽기 전용 프로필 카드다.
- `프로필 수정` 클릭 시 닉네임/주소 입력 폼과 `취소`, `프로필 저장` 버튼을 보여준다.
- 저장 성공 시 읽기 모드로 돌아가고 성공 토스트를 띄운다.
- 주소 수정 영역에는 위치 좌표가 초기화될 수 있다는 안내를 함께 노출한다.

## 제외 사항
- 주소 검색, 좌표 재계산, 브라우저 위치 연동은 `E18-2`에서 처리한다.
- 이메일 변경은 이번 카드 범위에 포함하지 않는다.
