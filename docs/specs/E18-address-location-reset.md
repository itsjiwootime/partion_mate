# E18 주소 및 위치 재설정 UX 스펙

- 작성일: 2026-03-18
- 대상 작업: `E18-2`
- 관련 화면: `frontend/src/pages/Profile.jsx`, `frontend/src/pages/Home.jsx`
- 관련 유틸: `frontend/src/utils/addressLocation.js`

## 목적
- 사용자가 프로필 주소를 바꾼 뒤 홈과 주변 지점 탐색 기준 위치를 다시 자연스럽게 맞출 수 있게 한다.
- 저장된 주소 좌표가 없는 상태에서도 홈 화면이 왜 기본 위치를 쓰는지 설명하고, 다음 행동을 안내한다.

## 프로필 편집 동작
- 프로필 수정 화면에 `주소 검색으로 위치 다시 설정` 버튼을 제공한다.
- 버튼 클릭 시 카카오 주소 검색을 열고, 선택한 주소를 카카오맵 geocode로 다시 좌표 변환한다.
- 주소 검색으로 선택한 주소와 좌표는 저장 요청에 함께 포함된다.
- 주소를 직접 입력하면 기존 좌표는 초기화되고, 저장 시 좌표 없이 저장된다.
- 편집 화면은 현재 저장된 기준 위치 좌표 또는 `저장된 위치 좌표 없음` 상태를 함께 노출한다.

## 홈 화면 안내
- 저장 좌표가 있으면 `내 주소 기준`으로 주변 지점을 조회한다.
- 저장 좌표가 없고 주소만 있으면 기본 위치를 사용하면서 프로필에서 위치를 다시 설정하라는 안내를 보여준다.
- 브라우저 위치로 전환한 경우, 현재는 임시 탐색 기준이며 버튼으로 주소 기준으로 돌아갈 수 있음을 안내한다.

## 제외 사항
- 브라우저 위치를 프로필 저장 좌표로 바로 영구 저장하는 기능은 이번 카드에 포함하지 않는다.
- 역지오코딩 기반 주소 자동 채우기는 다루지 않는다.
