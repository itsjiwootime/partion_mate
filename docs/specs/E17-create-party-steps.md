# E17 생성 폼 단계화 스펙

- 작성일: 2026-03-18
- 대상 작업: `E17-1`
- 관련 화면: `frontend/src/pages/CreateParty.jsx`
- 관련 API: `POST /party`

## 목적
- 파티 생성 입력을 한 화면에 모두 몰아넣지 않고, 사용자가 필요한 정보만 순서대로 입력하게 한다.
- 모바일에서도 긴 폼 스크롤과 누락 입력을 줄인다.
- 파티 생성 후 전용 채팅방이 자동으로 열리는 현재 구조에 맞춰 외부 오픈채팅 입력을 제거한다.

## 단계 구성
- 1단계 `기본 정보`
  - 지점
  - 파티 제목
  - 제품명
- 2단계 `소분 조건`
  - 총 가격
  - 총 수량
  - 호스트 요청 수량
  - 단위 표기
  - 최소 소분 단위
  - 보관 방식
  - 포장 방식
  - 포장재 제공 여부
  - 현장 소분 여부
- 3단계 `운영 정보`
  - 모집 마감 날짜
  - 모집 마감 시간
  - 거래 안내
  - 최종 확인 요약

## 진행 UI
- 상단에 현재 단계, 전체 단계 수, 완료 여부를 함께 보여준다.
- 현재 단계 이전 카드만 다시 눌러 이동할 수 있다.
- 하단에는 `이전 단계`, `다음 단계`, 마지막 단계에서는 `파티 생성하기` 버튼을 노출한다.

## 단계별 검증
- 1단계
  - 지점 선택 필수
  - 제품명 필수
  - 제목은 선택값이며 비워두면 제품명으로 대체
- 2단계
  - 총 가격 1원 이상
  - 총 수량 1개 이상
  - 호스트 요청 수량은 0 이상, 총 수량 이하
  - 단위 표기 필수
  - 최소 소분 단위 1 이상
- 3단계
  - 모집 마감 날짜/시간 모두 필수

## 최종 확인 요약
- 지점명
- 파티 제목
- 제품명
- 모집 마감 시각
- 총 가격
- 개당 예상 가격
- 호스트 예상 부담금
- 소분 기준
- 보관/포장 정보
- 거래 안내

## 제외 사항
- 카카오 오픈채팅 링크 입력은 제거한다.
- 초안 저장/복구는 이번 카드 범위에 포함하지 않는다.
- 비용 경고 문구와 상세 계산 UX 확장은 `E17-2`에서 보강한다.
