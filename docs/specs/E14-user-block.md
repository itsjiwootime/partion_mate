# E14 사용자 차단 API 및 상호작용 제한 스펙

- 작성일: 2026-03-18
- 대상 작업: `E14-2`
- 관련 API: `POST /api/blocks`, `DELETE /api/blocks/{targetUserId}`, `GET /api/blocks`

## 목적
- 사용자가 원치 않는 상대와의 후속 상호작용을 스스로 끊을 수 있게 한다.
- 현재 단계에서는 차단 이후 `파티 참여`, `채팅 진입`, `채팅 메시지 전송` 같은 새 상호작용을 우선 막는다.

## 차단 모델
- 차단은 방향성이 있다.
  - `blocker -> blocked`
- 같은 사용자를 중복 차단할 수 없다.
- 자기 자신은 차단할 수 없다.
- 차단 목록 조회는 로그인한 사용자가 직접 차단한 사용자만 반환한다.

## API

### `POST /api/blocks`

```json
{
  "targetUserId": 7
}
```

### `201 Created`

```json
{
  "id": 3,
  "targetUserId": 7,
  "targetUsername": "blocked-user",
  "createdAt": "2026-03-18T03:10:00",
  "createdAtLabel": "2026.03.18 03:10"
}
```

### `DELETE /api/blocks/{targetUserId}`
- 성공 시 `204 No Content`

### `GET /api/blocks`
- 로그인한 사용자가 차단한 사용자 목록을 최신순으로 조회한다.

## 상호작용 제한 규칙
- 파티 참여
  - 참여 요청 사용자와 현재 파티 참여자 사이에 차단 관계가 하나라도 있으면 참여를 막는다.
  - 오류 메시지: `차단한 사용자 또는 나를 차단한 사용자가 포함된 파티에는 참여할 수 없습니다.`
- 채팅 REST
  - 차단 관계가 있는 사용자가 포함된 파티 채팅방은 상세 조회, 읽음 처리, 메시지 전송, 공지 수정이 모두 막힌다.
  - 오류 메시지: `차단 관계가 있는 사용자가 포함된 채팅방에는 접근할 수 없습니다.`
- 채팅 STOMP
  - `/topic/chat/{partyId}` 구독과 `/app/chat/{partyId}/messages` 전송 모두 같은 차단 규칙을 적용한다.

## 현재 한계
- 기존에 이미 참여 중인 파티 구성원을 차단해도 자동 강퇴나 파티 탈퇴는 일어나지 않는다.
- 채팅방 목록은 아직 그대로 보이고, 실제 진입 시 에러 메시지로 제한 이유를 노출한다.
- 후기, 신고, 정산 같은 후속 기능의 차단 연계는 아직 적용하지 않는다.
