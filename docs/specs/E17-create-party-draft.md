# E17 생성 화면 초안 저장 및 복구 스펙

- 작성일: 2026-03-18
- 대상 작업: `E17-3`
- 관련 화면: `frontend/src/pages/CreateParty.jsx`
- 관련 유틸: `frontend/src/utils/createPartyDraft.js`

## 목적
- 파티 생성 도중 새로고침, 페이지 이탈, 탭 종료가 발생해도 작성 중인 내용을 다시 불러올 수 있게 한다.
- 단계형 생성 화면에서 사용자가 직전까지 입력하던 단계와 값을 함께 복구한다.

## 저장 위치
- 브라우저 `localStorage`를 사용한다.
- 저장 키는 로그인 사용자 기준으로 분리한다.
- 초안은 화면 입력값 전체와 현재 단계, 저장 시각, 스키마 버전, 작성 중인 `storeId`를 함께 저장한다.

## 저장 시점
- 사용자가 복구 배너에 응답한 뒤부터 자동 저장을 시작한다.
- 입력값이 초기 상태와 같아지면 초안을 저장하지 않고 기존 초안도 제거한다.
- 파티 생성 성공 시 초안을 즉시 제거한다.

## 복구 UX
- 저장된 초안이 있고 스키마가 유효하면 화면 상단에 복구 배너를 노출한다.
- 사용자는 `초안 복구` 또는 `새로 작성` 중 하나를 선택할 수 있다.
- 로그인 사용자가 달라지면 다른 사용자의 초안은 읽지 않는다.
- URL의 `storeId`가 있는 진입에서는 저장된 초안의 `storeId`가 같을 때만 복구 배너를 노출한다.
- `초안 복구`
  - 저장된 입력값과 단계로 화면을 복구한다.
- `새로 작성`
  - 저장된 초안을 삭제하고 현재 지점 기준의 초기값으로 시작한다.

## 스키마 및 만료 규칙
- 초안은 `version` 필드로 스키마 버전을 관리한다.
- 현재 버전과 다르면 자동으로 무시하고 삭제한다.
- 저장 후 7일이 지난 초안은 만료된 것으로 보고 자동 삭제한다.
- JSON 파싱 실패나 필드 구조가 맞지 않는 초안도 자동 삭제한다.

## 제외 사항
- 여러 초안을 동시에 저장하지 않는다.
- 서버 동기화는 이번 카드 범위에 포함하지 않는다.
