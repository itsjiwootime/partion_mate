# E17 생성 화면 예상 정산 및 참여 조건 미리보기 스펙

- 작성일: 2026-03-18
- 대상 작업: `E17-2`
- 관련 화면: `frontend/src/pages/CreateParty.jsx`
- 관련 유틸: `frontend/src/utils/createPartyPreview.js`

## 목적
- 호스트가 파티를 만들면서 비용과 수량 조건이 참여자에게 어떻게 보일지 즉시 이해하게 한다.
- 총 가격, 총 수량, 호스트 수량, 최소 소분 단위 사이의 관계를 한 화면에서 설명한다.
- 실제 정산은 이후 확정 값으로 다시 계산된다는 점을 유지하면서, 생성 시점에는 오해를 줄이는 예상치와 경고만 제공한다.

## 미리보기 항목
- 1단위 기준 예상 금액
- 최소 기준 참여 1명 예상 금액
- 호스트 예상 부담금
- 참여자에게 열리는 수량
- 최소 단위 기준 최대 참여자 수
- 참여 난이도 경고 메시지

## 계산 기준
- 참여자에게 열리는 수량 = `총 수량 - 호스트 수량`
- 1단위 기준 예상 금액
  - 기본값은 `총 가격 / 총 수량`의 내림값을 사용한다.
  - 나머지 금액이 있으면 최대값은 `기본값 + 1원`으로 표시한다.
- 최소 기준 참여 1명 예상 금액
  - 최소 소분 단위 수량을 기준으로 계산한다.
  - 나머지 금액이 있으면 최대 `최소 소분 단위`만큼 1원이 추가될 수 있는 범위로 표시한다.
- 호스트 예상 부담금
  - 호스트 수량 기준으로 같은 방식의 범위 값을 보여준다.
- 최소 단위 기준 최대 참여자 수
  - `floor(참여자에게 열리는 수량 / 최소 소분 단위)`로 계산한다.

## 경고 메시지 규칙
- 호스트 수량과 총 수량이 같으면 참여자 모집 불가 경고를 노출한다.
- 참여자에게 열리는 수량이 최소 소분 단위보다 작으면 모집이 어렵다는 경고를 노출한다.
- 최소 소분 단위 기준으로 나눴을 때 수량이 남으면 마지막 참여 수량 조정이 필요할 수 있다는 경고를 노출한다.
- 총 가격이 총 수량으로 정확히 나누어떨어지지 않으면 1원 단위 차이 가능성을 안내한다.

## 제외 사항
- 이 값들은 정산 확정 금액이 아니라 생성 화면용 예상치다.
- 참여 요청 입력 UX 자체는 이번 카드에서 바꾸지 않는다.
- 초안 저장/복구는 `E17-3`에서 처리한다.
