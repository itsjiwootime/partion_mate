# E14 노쇼 누적 정책 스펙

- 작성일: 2026-03-18
- 대상 작업: `E14-3`
- 관련 API: `POST /party/{id}/join`

## 목적
- 반복 노쇼 사용자의 새 파티 참여를 제한해 운영 리스크를 낮춘다.
- 완전 차단 전 한 차례 경고를 먼저 노출해 사용자에게 회복 기회를 준다.

## 누적 기준
- 노쇼 정책은 참여자(`PartyMemberRole.MEMBER`)의 확정 거래 이력만 본다.
- 누적 계산에는 `COMPLETED`, `NO_SHOW` 상태만 포함한다.
- 최근 확정 거래부터 역순으로 보면서 `COMPLETED`를 만나기 전까지의 연속 `NO_SHOW` 개수를 현재 누적으로 본다.
- 이 순서를 안정적으로 판단하기 위해 `PartyMember.tradeStatusUpdatedAt`을 함께 저장한다.

## 제한 단계
- 최근 연속 노쇼 `0회`
  - 제한 없음
- 최근 연속 노쇼 `1회`
  - 참여는 허용한다.
  - 응답의 `warning` 필드에 경고 코드를 함께 내려준다.
- 최근 연속 노쇼 `2회 이상`
  - 참여와 대기열 등록을 모두 막는다.
  - 오류 메시지: `최근 노쇼가 2회 연속 기록되어 새 거래를 1회 완료하기 전까지 파티에 참여할 수 없습니다.`

## 해제 조건
- 가장 최근 `NO_SHOW` 이후 `COMPLETED` 거래가 1회 기록되면 현재 누적은 `0`으로 본다.
- 과거 총 노쇼 횟수는 신뢰도 지표에 남아도, 참여 제한 정책은 해제된다.

## 참여 응답 확장

### 경고 없이 참여
```json
{
  "joinStatus": "JOINED",
  "message": "파티 참여가 완료되었습니다.",
  "waitingPosition": null,
  "party": {
    "id": 12
  },
  "warning": null
}
```

### 경고와 함께 참여
```json
{
  "joinStatus": "JOINED",
  "message": "파티 참여가 완료되었습니다.",
  "waitingPosition": null,
  "party": {
    "id": 12
  },
  "warning": {
    "code": "NO_SHOW_CAUTION",
    "message": "최근 노쇼가 1회 기록되어 있습니다. 한 번 더 노쇼가 누적되면 새 거래를 1회 완료하기 전까지 참여가 제한됩니다.",
    "activeNoShowCount": 1
  }
}
```

## 현재 한계
- 기존 신뢰도 요약은 총 노쇼 횟수를 계속 노출하므로, 참여 제한 단계와 수치가 다를 수 있다.
- 참여 제한은 새 참여 요청에만 적용되며, 이미 참여 중인 파티에는 소급 적용하지 않는다.
