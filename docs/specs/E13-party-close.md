# E13 호스트 취소 API 스펙

## 목적
- 호스트가 모집을 공식적으로 종료할 수 있게 한다.
- 자동 마감과 호스트 취소를 종료 사유로 구분한다.

## API

### `POST /party/{id}/close`
- 인증 필요
- 파티 호스트만 호출 가능
- 요청 본문 없음
- 응답
  - `200 OK`
  - `PartyDetailResponse`

## 종료 사유
- `DEADLINE_EXPIRED`
  - 마감 시간 경과로 자동 종료
- `HOST_CANCELED`
  - 호스트가 수동으로 파티를 종료

## 허용 조건
- 파티가 `CLOSED` 상태가 아니어야 한다.
- 마감 시간이 지나지 않아야 한다.
- 정산, 픽업, 송금, 거래가 시작되기 전이어야 한다.

## 금지 조건
- 참여자가 호출하는 경우
- 이미 종료된 파티
- 마감 시간이 지난 파티
- 아래 중 하나라도 진행된 파티
  - `actualTotalPrice` 확정
  - `pickupPlace`/`pickupTime` 확정
  - 참여자의 `paymentStatus`가 `PENDING` 이외 상태
  - 참여자의 `tradeStatus`가 `PENDING` 이외 상태
  - 참여자의 픽업 확인 완료

## 후속 동작
- 파티 상태를 `CLOSED`로 바꾸고 `closeReason=HOST_CANCELED`를 기록한다.
- 참여자에게 종료 알림을 발송한다.
  - 호스트 본인에게는 자기 취소 알림을 보내지 않는다.
- 채팅 시스템 메시지를 남긴다.
- 실시간 `party-updated` 이벤트를 발송한다.

## 오류 응답 예시
- `403 BUSINESS_ERROR`
  - `파티 종료는 호스트만 처리할 수 있습니다.`
- `409 BUSINESS_ERROR`
  - `이미 종료된 파티입니다.`
  - `마감 시간이 지나 종료된 파티입니다.`
  - `정산, 픽업, 송금 또는 거래가 시작된 파티는 호스트 취소할 수 없습니다.`
