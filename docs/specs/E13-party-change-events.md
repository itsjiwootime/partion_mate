# E13 파티 변경 이벤트 동기화 스펙

- 작성일: 2026-03-18
- 대상 작업: `E13-3`
- 관련 API: `PUT /party/{id}`, `POST /party/{id}/close`, `GET /party/stream`

## 목적
- 호스트의 파티 수정과 취소가 실시간 화면, 앱 내 알림, 채팅 시스템 메시지에 같은 의미로 전달되게 한다.
- 조건 변경 이후에도 참여자 화면과 알림이 같은 의미로 갱신되게 한다.

## 파티 수정 동기화 흐름
1. 호스트가 `PUT /party/{id}`를 호출한다.
2. 서버는 수정 전 스냅샷과 수정 후 상태를 비교해 변경 항목 요약을 만든다.
3. 채팅방에는 `호스트가 파티 조건을 수정했습니다. 변경 항목: ...` 시스템 메시지를 남긴다.
4. 기존 참여자(호스트 제외)에게 `PARTY_UPDATED` outbox 이벤트를 발행한다.
5. SSE 스트림에는 최종 상태 스냅샷을 `realtimeTrigger=PARTY_UPDATED`로 전송한다.

## `PARTY_UPDATED` outbox payload

```json
{
  "partyId": 12,
  "partyTitle": "코스트코 비타민 소분",
  "changeSummary": "변경 항목: 총 수량, 마감 시간, 안내 문구",
  "joinedUserIds": [21]
}
```

## 알림 규칙
- 참여자 알림
  - 타입: `PARTY_UPDATED`
  - 제목: `파티 조건이 변경되었습니다`
  - 메시지: `{partyTitle} 파티 조건이 변경되었습니다. {changeSummary}`
- 호스트 본인에게는 자기 수정 알림을 보내지 않는다.

## 프론트 동기화 규칙
- 파티 목록, 파티 상세, 내 파티는 기존 SSE 전략을 유지한다.
- 채팅 화면은 전역 파티 스트림을 구독하고, 현재 채팅방 또는 내 채팅방 목록에 포함된 파티 ID가 변경되면 방 목록/상세 API를 다시 조회한다.
- SSE 연결이 끊기면 화면별 기존 fallback 재조회 전략을 그대로 사용한다.
