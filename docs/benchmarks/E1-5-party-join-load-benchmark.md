# E1-5 Party Join Load Benchmark

## 측정 조건
- 실행 일시: 2026-03-27
- 실행 명령: `./scripts/run_party_join_load_benchmark.sh`
- 측정 대상: `PartyService.joinParty`
- 프로필: `test`
- DB: H2 in-memory
- 시나리오:
  - 총 동시 요청 300건
  - 파티 총 수량 20개
  - 호스트 선점 수량 1개
  - 일반 사용자 요청 수량은 모두 1개

## 측정 결과
- 즉시 참여 성공: 19건
- 잔여 수량 부족으로 거절: 281건

- 중복 참여: 0건

- P95 응답시간: 196.30ms
- P99 응답시간: 382.89ms

## 해석
- 비관적 락과 즉시 실패 정책 적용 후에도 총수량 20개를 초과하지 않았다.
- 남은 수량을 초과한 요청은 모두 `409 CONFLICT` 성격의 비즈니스 예외로 정리됐다.
- 현재 수치는 서비스 계층 + H2 기준 내부 벤치마크이며, HTTP 계층과 MySQL 락 비용은 반영하지 않는다.

## 재실행 메모
- 수치는 실행 환경에 따라 달라질 수 있으므로, 이력서에는 측정 조건과 함께 기재한다.
- 이후 MySQL/Docker 환경 벤치마크를 추가하면 운영 유사 수치로 보강할 수 있다.
