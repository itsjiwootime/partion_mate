# ADR-20260317-mobile-chat-ux-policy

- 날짜: 2026-03-17
- 상태: Accepted
- 관련 작업: `E9-4`
- 관련 파일: `frontend/src/pages/Chat.jsx`

## 배경
- 채팅 화면은 데스크톱 레이아웃을 그대로 모바일에 적용해, 방을 선택해도 목록과 대화창이 세로로 길게 이어져 대화에 집중하기 어려웠다.
- 새 메시지가 도착해도 스크롤이 최신 메시지로 이동하지 않아 실시간 채팅 체감이 약했다.
- 연결이 아직 준비되지 않은 상태에서 전송 버튼을 눌러도 별도 피드백이 없어 사용자가 왜 전송이 안 되는지 알기 어려웠다.

## 결정사항
- 모바일에서는 채팅방 선택 후 목록을 숨기고 대화 영역만 보여주는 단일 뷰 레이아웃을 사용한다. 데스크톱에서는 기존 2열 레이아웃을 유지한다.
- 메시지 목록 컨테이너에 ref를 두고, 메시지 변경 시 `requestAnimationFrame`으로 맨 아래로 자동 스크롤한다.
- 채팅방 목록 조회 실패와 상세 조회 실패에는 각각 재시도 버튼을 노출한다.
- 연결이 아직 준비되지 않은 상태에서 메시지 전송을 시도하면 토스트로 오류를 안내한다.

## 대안
- 모바일에서도 목록과 상세를 같은 화면에 계속 노출
  - 장점: 구현이 단순하다.
  - 단점: 대화창이 아래로 밀려 핵심 사용 흐름이 불편하다.
- 자동 스크롤 없이 사용자가 직접 최신 메시지로 이동
  - 장점: 스크롤 위치 보존이 단순하다.
  - 단점: 실시간 채팅 UX가 떨어지고 새 메시지를 놓치기 쉽다.
- 별도 모바일 채팅 전용 페이지를 추가
  - 장점: 플랫폼별 레이아웃을 완전히 분리할 수 있다.
  - 단점: 현재 프로젝트 규모 대비 라우팅과 상태 관리가 불필요하게 늘어난다.

## 결과
- 긍정적
  - 모바일에서 채팅방 선택 후 대화에 집중하기 쉬워졌다.
  - 새 메시지 수신 시 최신 메시지를 바로 볼 수 있어 실시간 체감이 좋아졌다.
  - 조회 실패와 연결 미준비 상태에 대한 사용자 피드백이 추가됐다.
- 부정적
  - 자동 스크롤은 긴 과거 메시지를 읽는 중일 때 위치를 다시 아래로 보낼 수 있다.
  - 모바일/데스크톱 분기 조건이 늘어 채팅 화면 JSX가 조금 복잡해졌다.
- 중립적
  - 이후 메시지 수가 더 많아지면 "새 메시지로 이동" 버튼이나 스크롤 위치 보존 정책을 추가로 검토할 수 있다.
