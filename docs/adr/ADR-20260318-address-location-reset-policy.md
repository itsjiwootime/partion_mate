# ADR-20260318-address-location-reset-policy

- 날짜: 2026-03-18
- 상태: Accepted
- 관련 작업: `E18-2`
- 관련 파일: `frontend/src/pages/Profile.jsx`, `frontend/src/pages/Home.jsx`, `frontend/src/utils/addressLocation.js`

## 배경
- `E18-1`에서 프로필 주소 수정은 가능해졌지만, 사용자가 주소를 직접 바꾸면 저장 좌표가 비워질 수 있어 홈 화면이 기본 위치를 쓰게 된다.
- 이 상태를 설명하지 않으면 사용자는 왜 주변 지점이 원하는 동네 기준으로 나오지 않는지 이해하기 어렵다.
- 회원가입에는 이미 주소 검색과 좌표 변환 흐름이 있지만, 프로필 수정에는 같은 수준의 위치 재설정 UX가 없어 설정 경험이 단절돼 있었다.

## 결정사항
- 프로필 수정 화면에서 카카오 주소 검색과 geocode를 이용해 주소와 좌표를 함께 다시 저장할 수 있게 한다.
- 주소를 수동 입력하면 좌표를 초기화하고, 사용자가 주소 검색을 다시 수행하지 않으면 좌표 없는 상태로 저장한다.
- 홈 화면은 저장 좌표 유무와 브라우저 위치 사용 여부에 따라 현재 탐색 기준을 설명하는 안내 문구를 보여준다.
- 주소 검색과 geocode 호출은 `addressLocation` 유틸로 분리해 프로필 위치 재설정 흐름에서 재사용한다.

## 대안
- 프로필 편집에서도 주소 텍스트만 수정하고 좌표 재설정은 별도 처리하지 않는 방식
  - 장점: 구현이 가장 단순하다.
  - 단점: 홈 탐색 기준이 기본 위치로 바뀌는 이유를 사용자가 이해하기 어렵고, 위치 일관성이 떨어진다.
- 브라우저 위치를 바로 프로필 저장 좌표로 쓰는 방식
  - 장점: 현재 위치를 빠르게 저장할 수 있다.
  - 단점: 주소 문자열과 저장 좌표가 어긋날 수 있고, 역지오코딩 없이 저장하면 사용자 이해가 나빠진다.
- 백엔드에서 주소 문자열만 받으면 서버가 좌표를 계산하는 방식
  - 장점: 프론트 로직이 단순해진다.
  - 단점: 현재는 카카오 SDK 기반 프론트 흐름이 이미 존재하고, 서버 외부 연동까지 확장해야 한다.

## 결과
- 긍정적 트레이드오프: 사용자가 프로필에서 주소와 탐색 기준 위치를 다시 맞출 수 있고, 홈이 현재 기준 위치를 더 명확하게 설명한다.
- 부정적 트레이드오프: 카카오 주소 검색과 지도 SDK 의존이 프로필 화면에도 들어가 프론트 외부 스크립트 의존성이 늘어난다.
- 중립적 트레이드오프 또는 후속 영향 사항: 브라우저 위치를 영구 저장하거나 역지오코딩으로 주소를 자동 보정하는 기능은 후속 카드로 남는다.
