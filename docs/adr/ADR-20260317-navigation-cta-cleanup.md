# ADR-20260317-navigation-cta-cleanup

- 날짜: 2026-03-17
- 상태: Accepted
- 관련 작업: `E9-5`
- 관련 파일: `frontend/src/components/Header.jsx`, `frontend/src/pages/Home.jsx`

## 배경
- 헤더의 `가까운 지점` 버튼과 사용자 배지는 눈에 보이지만 실제 이동 동작이 연결되지 않아 기대를 만들고 끝나는 CTA였다.
- 홈 화면의 `거리순`은 버튼처럼 보였지만 정렬 동작을 바꾸지 않는 데드 CTA였다.
- 실제 동작이 없는 CTA는 기능 부족보다 더 큰 신뢰 하락을 만든다.

## 결정사항
- 헤더의 `가까운 지점` 버튼은 홈(`/`)으로, 사용자 배지는 프로필(`/me`)로 이동하게 연결한다.
- 홈의 `거리순`은 클릭 가능한 버튼이 아니라 현재 정렬 상태를 설명하는 배지로 바꾼다.
- 공통 탐색에서는 아직 지원하지 않는 동작을 CTA로 노출하지 않고, 현재 제공되는 핵심 이동 경로만 유지한다.

## 대안
- 기존 CTA를 유지하고 추후 기능 구현 예정으로 둔다
  - 장점: 나중에 기능을 붙일 여지를 남긴다.
  - 단점: 현재 사용자에게는 고장난 버튼처럼 보인다.
- 거리 정렬 버튼을 눌러도 현재와 동일한 동작만 수행
  - 장점: 클릭 이벤트는 생긴다.
  - 단점: 사용자 기대와 결과가 달라 UX 문제가 해결되지 않는다.
- 공통 헤더 CTA를 모두 제거
  - 장점: 혼란이 줄어든다.
  - 단점: 홈/프로필로 가는 빠른 이동 경로도 함께 사라진다.

## 결과
- 긍정적
  - 화면에 보이는 주요 CTA가 실제 동작을 수행하게 됐다.
  - 홈 정렬 상태를 설명형 UI로 바꿔 불필요한 클릭 기대를 줄였다.
  - 공통 헤더의 이동 경로가 명확해졌다.
- 부정적
  - 지금은 헤더에서 제공하는 이동 경로가 단순해서 향후 탐색 기능 확장 여지는 줄어든다.
  - 거리 정렬 방식을 바꾸는 상호작용 자체는 아직 제공하지 않는다.
- 중립적
  - 이후 필터/정렬 기능이 실제로 생기면 다시 버튼형 UI로 확장할 수 있다.
