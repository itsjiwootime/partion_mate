# ADR-20260318-party-host-close-policy

- 날짜: 2026-03-18
- 상태: Accepted
- 관련 작업: `E13-2`
- 관련 파일: `src/main/java/com/project/partition_mate/service/PartyService.java`, `src/main/java/com/project/partition_mate/service/PartyLifecycleService.java`, `src/main/java/com/project/partition_mate/domain/PartyCloseReason.java`

## 배경
- 기존 종료 모델은 `DEADLINE_EXPIRED`만 지원해서 호스트가 모집을 중단하고 싶을 때 사용할 공식 흐름이 없었다.
- 이 때문에 호스트는 참여 취소 API를 사용할 수도 없고, 실제로는 채팅이나 안내문으로만 비공식 취소를 공지해야 했다.
- 반면 정산, 픽업, 송금, 거래가 진행된 뒤에 호스트 취소를 무제한 허용하면 환불, 분쟁, 신뢰도 정책이 얽히면서 운영 복잡도가 크게 올라간다.

## 결정사항
- 호스트 취소는 `POST /party/{id}/close` API로 제공하고, 파티 호스트만 호출할 수 있게 한다.
- 종료 사유에 `HOST_CANCELED`를 추가해 자동 마감(`DEADLINE_EXPIRED`)과 구분한다.
- 호스트 취소는 `RECRUITING` 또는 `FULL` 상태이면서 마감 시간이 지나지 않은 파티에만 허용한다.
- 정산, 픽업, 송금, 거래가 시작된 파티는 호스트 취소를 금지한다.
  - 구체적으로 `actualTotalPrice`, `pickupSchedule`, 참여자 `paymentStatus`, `tradeStatus`, `pickupAcknowledged` 중 하나라도 진행되면 취소 불가로 본다.
- 호스트 취소 시 대기열 `WAITING` 항목은 모두 `EXPIRED`로 종료하고, 참여자와 대기자에게는 기존 `PARTY_CLOSED` outbox 이벤트를 재사용해 알림을 발송한다.
- 단, 호스트 본인에게는 자기 취소 알림을 보내지 않는다.
- 채팅에는 `호스트가 파티를 취소해 모집이 종료되었습니다.` 시스템 메시지를 남기고, 실시간 이벤트는 기존 `PARTY_CLOSED` trigger를 재사용한다.

## 대안
- 호스트 취소를 별도 종료 사유 없이 `DEADLINE_EXPIRED`와 같은 흐름으로 처리한다.
  - 장점: enum, 메시지, 알림 처리가 단순하다.
  - 단점: 사용자가 종료 원인을 구분할 수 없고, 운영 로그와 사용자 커뮤니케이션 품질이 떨어진다.
- 호스트 취소를 언제든 허용하고 후속 환불 처리는 나중에 해결한다.
  - 장점: 호스트 자유도가 높고 구현이 빠르다.
  - 단점: 정산/거래 시작 후 취소 시 환불 책임과 분쟁 처리가 불명확해져 운영 리스크가 커진다.
- 호스트 취소 시 파티를 `delete`로 제거한다.
  - 장점: 데이터 모델은 단순해 보인다.
  - 단점: 후기, 알림, 채팅, 실시간 상태, 감사 추적이 모두 끊겨 운영 이력 보존에 불리하다.

## 결과
- 긍정적 트레이드오프를 작성한다.
  - 호스트가 공식 API로 파티를 종료할 수 있어 운영 흐름이 명확해진다.
  - 자동 마감과 호스트 취소를 종료 사유로 분리해 알림과 화면에서 더 정확한 메시지를 제공할 수 있다.
  - 거래가 시작되기 전까지만 취소를 허용해 환불/분쟁 부담을 초기 단계에서 줄일 수 있다.
- 부정적 트레이드오프를 작성한다.
  - 거래 시작 여부 판단 규칙이 늘어나면서 종료 전 검증 로직이 복잡해진다.
  - 호스트가 픽업이나 정산을 먼저 눌러버린 뒤 취소를 원해도 별도 운영 수동 절차가 필요하다.
  - 현재는 취소 사유 상세 메모나 환불 워크플로까지 다루지 않으므로 후속 카드가 필요하다.
- 중립적 트레이드오프 또는 후속 영향 사항을 작성한다.
  - 이후 신고/차단, 외부 알림, 관리자 도구가 들어오면 `HOST_CANCELED` 종료 사유를 운영 지표에도 연결할 수 있다.
  - 추후 환불 워크플로가 생기면 `진행 시작 후 취소 금지` 정책을 일부 완화할지 다시 판단할 수 있다.
