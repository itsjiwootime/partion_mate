# ADR-20260318 Trust Signal Emphasis Policy

## 배경
- 신뢰도 점수와 후기/완료율/노쇼 데이터는 이미 계산되고 있었지만, 사용자가 빠르게 위험 신호를 파악하기에는 표현이 평면적이었다.
- 같은 데이터를 파티 상세와 프로필에서 다르게 보여주면 해석 기준이 달라질 수 있어 공통 표현 규칙이 필요했다.

## 결정사항
- `trustLevel`, `reviewCount`, `completionRate`, `noShowCount`를 공통 유틸에서 해석해 배지, 하이라이트, 경고 문구를 생성한다.
- 파티 상세와 프로필 모두 같은 배지 라벨과 같은 경고 조건을 사용한다.
- 위험 신호가 없을 때도 빈 상태 대신 안정 메시지를 보여줘 사용자가 신뢰 판단을 빠르게 끝낼 수 있게 한다.

## 대안
- 단순 점수와 등급만 크게 보여주는 방안
  - 장점: 구현이 가장 단순하다.
  - 단점: 왜 주의해야 하는지와 어떤 수치가 좋은지 한눈에 파악하기 어렵다.
- 화면마다 다른 기준으로 경고 문구를 다르게 두는 방안
  - 장점: 각 화면 문맥에 맞춘 개별 최적화가 가능하다.
  - 단점: 같은 사용자라도 화면마다 인상이 달라져 일관성이 무너진다.
- 노쇼만 따로 경고하고 완료율은 숨기는 방안
  - 장점: 강한 위험 신호만 강조해 시각적 부담이 적다.
  - 단점: 완료율이 낮은 사용자에 대한 중간 위험 신호를 놓치게 된다.

## 결과
- 긍정적: 사용자는 배지와 경고 문구만 보고도 거래 상대의 최근 상태를 빠르게 이해할 수 있다.
- 부정적: 임계값 기반 경고는 데이터가 적은 신규 사용자에게 다소 보수적으로 보일 수 있다.
- 중립적: 현재 규칙은 프론트 표시 정책이며, 실제 차단/참여 제한 같은 운영 정책과는 분리되어 있다.
